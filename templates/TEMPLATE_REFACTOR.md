# TEMPLATE — 重構任務交辦範本
# 用途：交辦「改結構不改行為」類任務給 MID 模型。
# 重構的鐵律：對外行為零改變。這是與實作任務的本質區別。

═══════════════════════════════════════════

在執行任何動作前，先完成加載程序：
讀取 CLAUDE.md、rules_core/01_WEAK_MODEL_CORRECTIONS.md、
rules_core/02_DISPATCH_PROTOCOL.md、rules_core/03_JUDGMENT_CHECKLIST.md。
你的第一行輸出必須是加載確認行，第二行必須是任務卡。
未輸出這兩行，禁止動任何檔案。

---

TASK: 重構 — [重構什麼，一句話]
動機：[為什麼重構：可讀性/效能/解耦/準備新功能]

重構範圍：
- 目標檔案：[精確清單]
- 凍結區：[明確禁止動的檔案/模組]

行為凍結證明（重構前必做，這是第 0 步）：
1. 先跑 [測試指令]，記錄當前全部通過的測試清單作為基線。
2. 若目標程式碼無測試覆蓋 → 停止，輸出
   「HUMAN GATE: 重構無測試保護，先補測試還是接受風險？」
   禁止在無測試保護下靜默重構。

DONE-WHEN（全部滿足才算完成）：
- [ ] 基線測試 100% 依然通過（一個都不能少）
- [ ] 對外介面零改變：函式簽名、API 路徑、回傳格式、CLI 參數
      全部與重構前完全一致（有意的介面變更 = 這不是重構，
      是改版，退回重新開任務）
- [ ] [重構目標的可判定驗收：如「該函式從 120 行拆為 <40 行
      的子函式」「循環依賴消除，import graph 無環」]
- [ ] 無殘留：無被遺棄的死程式碼、無空檔案

MUST-NOT：
- 禁止在重構中「順手」修 bug。發現 bug → 記錄，開新任務。
  （bug 修復會改變行為，污染「行為凍結」驗證。）
- 禁止在重構中「順手」加新功能。
- 禁止一次性大爆炸重寫。步驟必須是：小步改 → 跑測試 →
  綠了再下一步。每個小步都可獨立回退。
- 改檔禁用省略符（01 檔規則 3.2）。

熔斷提醒：
測試變紅且 2 次嘗試未恢復 → 全部回退到基線，輸出升級包。
判斷訊號 D2：重構的改動量若超過原程式碼的 2 倍 → PATH DEAD，
方向錯誤，停止並回報。

REPORT FORMAT（完工時逐項填寫）：
RESULT: DONE / BLOCKED / PARTIAL
CHANGED: [動過的檔案清單]
BASELINE: [基線測試數] → AFTER: [重構後通過數]（必須相等）
INTERFACE-DIFF: none（必須為 none，否則任務失敗）
VERIFIED-BY: [自檢 + 等待獨立驗證聲明]
DEVIATION: [偏差申報，無則 none]
NEXT: [建議下一步，或 none]
