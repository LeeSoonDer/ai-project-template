# Debugging

## Use When
- There is an error, failing test, broken UI, or unexpected behavior.

## Do Not Use When
- The user only asks for conceptual explanation.

## Workflow
1. Read relevant project memory.
2. Inspect relevant files or docs.
3. Do the smallest useful action.
4. Report concise result and next step.
