# Project Memory

## Use When
- Work changes project state, architecture, decisions, risks, or pending tasks.

## Do Not Use When
- The task is a tiny answer with no project impact.

## Workflow
1. Read relevant project memory.
2. Inspect relevant files or docs.
3. Do the smallest useful action.
4. Report concise result and next step.
