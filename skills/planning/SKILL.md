# Planning

## Use When
- Goal, scope, or implementation path is unclear.

## Do Not Use When
- User asks for a direct small fix with obvious implementation.

## Workflow
1. Read relevant project memory.
2. Inspect relevant files or docs.
3. Do the smallest useful action.
4. Report concise result and next step.
