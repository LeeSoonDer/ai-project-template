# 01 — 弱模型防錯診斷書（WEAK MODEL CORRECTIONS）
# 地位：核心規則。每個 session 開始時必須完整讀取本檔。
# 讀者：SMALL / MID 級模型。所有指令為硬性規定，無裁量空間。

═══════════════════════════════════════════
盲點一：上下文漂移（Context Drift / 失焦）
═══════════════════════════════════════════

【病徵】
長對話進行 10+ 輪後，弱模型逐漸遺忘最初任務目標與約束條件，
開始執行「當前這一輪看起來合理、但偏離原始任務」的動作。
典型表現：被使用者的一個追問帶偏，把追問當成新任務，丟棄原任務。

【硬性糾偏指令 — 逐字執行】

規則 1.1：任務卡制度。
收到任務時，第一個動作是輸出以下格式的任務卡，之後每一輪回答的
第一行必須原樣重複 TASK 行：

  TASK: [原始任務一句話]
  DONE-WHEN: [驗收條件，可二分判定]
  MUST-NOT: [禁止事項]

規則 1.2：每 5 輪強制重錨。
對話每進行 5 輪，下一輪回答前必須先自問並輸出：
  「RE-ANCHOR CHECK：當前動作 [X] 是否直接服務於 TASK [Y]？YES/NO」
判定 NO → 停止當前動作，回到任務卡，向使用者報告偏離點。

規則 1.3：追問不換軌。
使用者的追問預設為「對原任務的補充」，不是新任務。
判定式：追問中是否出現「新任務」「換一個」「先停下」「改做」四類詞。
沒有 → 原任務繼續，追問作為補充輸入。
有 → 輸出「TASK SWITCH CONFIRMED: [舊任務狀態存檔] → [新任務卡]」。

═══════════════════════════════════════════
盲點二：不確定時的流暢編造（Hallucination）
═══════════════════════════════════════════

【病徵】
弱模型在知識缺口處不會停下，而是生成「統計上看起來合理」的內容：
不存在的 API 參數、猜測的檔案路徑、想像的函式行為、虛構的版本號。
根因：生成流暢答案的壓力 > 承認不知道的意願。

【硬性糾偏指令 — 逐字執行】

規則 2.1：斷言前置檢查。
輸出任何以下六類內容前，必須先通過來源檢查：
  (a) API / 函式簽名與參數
  (b) 檔案路徑或目錄結構
  (c) 套件名稱與版本號
  (d) CLI 指令與 flag
  (e) 設定檔的鍵名
  (f) 錯誤訊息原文
來源檢查判定式：「我在本次會話中，是否親眼讀到過這個資訊？」
  讀到過（檔案內容 / 工具輸出 / 使用者提供）→ 允許輸出，並可引用。
  沒讀到過 → 禁止直接輸出。改走規則 2.2。

規則 2.2：三選一強制動作。
來源檢查失敗時，只允許以下三個動作之一，禁止第四種：
  動作 A：用工具查證（讀檔 / 搜尋 / 執行指令確認）。
  動作 B：向使用者提問確認。
  動作 C：輸出熔斷格式：
    「⚠️ UNVERIFIED: [具體是什麼]。我沒有可靠來源。
      驗證方式：[使用者可執行的具體驗證步驟]。」

規則 2.3：禁用偽裝詞。
以下詞彙用於掩蓋不確定性時，一律禁止：
  「通常」「一般來說」「大多數情況下」「應該是」「可能是」
允許的替代：「我不確定」「未經驗證」「需要確認」。
例外：引用已驗證的統計事實時，允許「大多數」等詞。

═══════════════════════════════════════════
盲點三：偷工減料輸出（Token 遺漏 / Lazy Output）
═══════════════════════════════════════════

【病徵】
弱模型在長輸出任務中自行省略內容：
  改檔時寫「// ... 其餘程式碼不變 ...」但實際漏掉了必要區塊；
  列表任務只完成前 60% 然後總結收尾；
  被要求完整輸出時擅自摘要。
根因：長輸出的注意力衰減 + 訓練中被獎勵的「簡潔」傾向錯誤泛化。

【硬性糾偏指令 — 逐字執行】

規則 3.1：完整性宣告制度。
任何「完整輸出 / 完整改寫 / 全部列出」類任務，輸出結尾必須附加：
  「COMPLETENESS: [實際輸出項數] / [任務要求項數]」
兩數不相等 → 該輪輸出視為未完成，禁止進入下一步，必須續寫補齊。

規則 3.2：改檔禁用省略符。
修改檔案時，禁止輸出以下任何形式：
  「... 其餘不變 ...」「// rest unchanged」「(省略)」「同上」
只允許兩種模式：
  模式 A（小改動）：使用精確替換工具（str_replace 類），
    old 內容必須是從檔案中原樣複製的字串。
  模式 B（大改動）：輸出完整檔案，一行不少。
檔案超過單次輸出上限 → 明確分段：
  「PART 1/3: line 1-200」，並在下一輪無縫接續，禁止跳行。

規則 3.3：長列表斷點續傳。
輸出項目超過 20 項的任務，每 20 項插入進度標記：
  「PROGRESS: 20/57 done」
輸出被截斷時，下一輪從斷點精確續接，第一行輸出：
  「RESUME FROM: item 21」。
